## Instructions to Participants

*Read aloud or provide as handout:*

> You will review 5 refactoring suggestions generated by an automated tool.
> Each suggestion proposes extracting a group of methods from a large Java
> class ("God Class") into a new, focused class.
>
> For each suggestion, you will see:
> - The original class name and size
> - Which methods are proposed for extraction
> - The suggested name for the new class
> - The tool's rationale for the extraction
>
> All suggestions have been verified to compile and pass the project's
> existing test suite. Your task is to evaluate whether the suggestion
> is a GOOD refactoring from a software design perspective.
>
> There are no right or wrong answers. We want your honest professional
> judgment.

## Suggestion 1 of 5: DateUtils → DateParser

**Source project**: Apache Commons Lang
**Original class**: `DateUtils` (67 methods, 12 fields)
**Verification**: Compiles (Maven) ✓ | Tests pass ✓

**Methods to extract (5)**:
- `parseDate(String, Locale, String[])`

- `parseDateStrictly(String, Locale, String[])`
- `parseDateStrictly(String, String[])`
- `parseDateWithLeniency(String, Locale, String[], boolean)`

## For the Researcher — Data Collection Notes

### Counterbalancing
- Randomize the order of 5 suggestions for each participant
- In Q3b naming comparison, randomize which name is "A" vs "B"
- Half participants see GenEC name as A, half as B

### Expected Analysis
- Per-question Likert means +/- SD
- Naming preference: proportion choosing GenEC vs auto-name
- Inter-rater agreement: Krippendorff's alpha or Fleiss' kappa
- Wilcoxon signed-rank for paired naming comparison
- Thematic analysis on Q5/Q10 free text

### Sample Size Justification
- 15-20 participants provides sufficient power for:
  - Detecting medium effect size (d=0.5) on Likert scales
  - 80% power with alpha=0.05 (Wilcoxon)
  - Meaningful proportions for naming preference (binomial CI)
